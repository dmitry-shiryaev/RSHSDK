Detailed Description

Initialization structure for signal generators.

This structure is used for GSPF series devices.
Corresponding caps codes are RSH_CAPS_SOFT_GENERATION_IS_AVAILABLE and RSH_CAPS_SOFT_INIT_GSPF.

See Also
RshInitDAC
Examples:
GSPF-052PCI_Generate.cpp, and GSPF-053USB_Generate.cpp.

enum RshInitGSPF::Control

List of additional flags.

Use this flags to specify additional options in RshInitGSPF::control field.

Enumerator
FilterOff 

Turn OFF filter.

Mutually exclusive with RshInitGSPF::FilterOn

PlayOnce 

Play buffer only once.

Signal in buffer will be played once, and then DAC will be stopped.
Mutually exclusive with RshInitGSPF::PlayLoop.

SynchroFront 

Start generation on trigger front.

Only valid when RshInitGSPF::External is used as RshInitGSPF::startType value.
Generation will be started on trigger front. Mutually exclusive with RshInitGSPF::SynchroDecline.

SynthesizerOff 

Frequency synthesizer is not used.

DAC will be sampled using Quartz frequency and devider.
Mutually exclusive with RshInitGSPF::SynthesizerOn.

SynthesizerOn 

Frequency synthesizer is used.

DAC will be sampled by frequency synthesizer.
Mutually exclusive with RshInitGSPF::SynthesizerOff.

FilterOn 

Turn ON filter.

Turn on low pass filter with cut edge at 10.7MHz Mutually exclusive with RshInitGSPF::FilterOff

PlayLoop 

Play buffer continuously.

Signal in buffer will be played continuously.
Mutually exclusive with RshInitGSPF::PlayOnce.

Remarks
When using this flag, signal will be played even when your program is no more running. Call IRshDevice::Stop() to stop playing.
SynchroDecline 

Start generation on trigger decline.

Only valid when RshInitGSPF::External is used as RshInitGSPF::startType value.
Generation will be started on trigger decline. Mutually exclusive with RshInitGSPF::SynchroFront.

enum RshInitGSPF::StartType

Start mode list.

This enum entities can be used to specify how to start generation process.
To do so, set one of them as value for RshInitGSPF::startType field.

Enumerator
Program 

Program start.

Generation will start as soon as command will be acquired by the device.
That means, DAC will start as soon as IRshDevice::Start() method is called.

Remarks
"As soon" is not immediately, because there is always some delays for functions calls, data transfer and device responce.
External 

Start signal generation on external trigger.

Use external trigger input as source for signal generation start command.

FrequencyExternal 

Use external frequency source for sampling.

External frequency will be used for sampling DAC.
External input is used as sampling source for DAC. See user manual for details about sampling signal parameters.

Member Data Documentation

U32 RshInitGSPF::attenuator

Attenuator value.

One of the RshInitGSPF::Attenuator enum values. Predifined set of attenuation level constants can be used.
By default, signal is not attenuated.

Remarks
Attenuation is applied to signal just before output, that means, that using attenuator one can improve DAC dynamic range for small amplitude signals generated.
Examples:
GSPF-052PCI_Generate.cpp, and GSPF-053USB_Generate.cpp.

Referenced by operator<<(), operator=(), operator==(), and RshInitGSPF().

U32 RshInitGSPF::control

Additional flags.

'OR' Combination of RshInitGSPF::Control enum elements. This field is used to specify additional signal generation options, such as using of frequency synthesizer, filter and so on.

Remarks
By default, this parameter is equal to zero, which corresponds to following flag combination:
RshInitGSPF::SynthesizerOff | RshInitGSPF::FilterOff | RshInitGSPF::PlayOnce | RshInitGSPF::SynchroFront
