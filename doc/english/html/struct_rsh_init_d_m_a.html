Detailed Description

Initialization structure for DMA-type devices.

This structure usually is used for low-frequency (10MHz or less) devices. One can use RSH_CAPS_SOFT_INIT_DMA to find if device abstraction library supports this structure or not.
DMA-type device features:
low-frequency, can work in continious data acquisition moode, do not have onboard memory (usually), do not have many synchronization possibilities.

Remarks
DMA name is used for historical reasons, some devices do not have actual DMA feature (all USB devices, for examples).
Examples:
FrameFrequencyMode_EN.cpp, IRshDevice_Get_RSH_GET_WAIT_GATHERING_COMPLETE_EN.cpp, IRshDevice_Init_RshInitDMA_EN.cpp, LA-1.5PCI_Persistent.cpp, LA-1.5PCI_StartStop.cpp, LA-20USB_Persistent.cpp, LA-20USB_StartStop.cpp, LA-2M5PCI_Persistent.cpp, LA-2M5PCI_StartStop.cpp, LA-2USB_Persistent.cpp, LA-2USB_StartStop.cpp, LA-50USB_Persistent.cpp, LA-50USB_StartStop.cpp, LA-5Net_Persistent.cpp, LA-5Net_StartStop.cpp, LA-i24USB_Persistent.cpp, LA-i24USB_StartStop.cpp, LA-n150PCI_Persistent.cpp, LA-n150PCI_StartStop.cpp, Leonardo2_Persistent.cpp, and Leonardo2_StartStop.cpp.

Member Enumeration Documentation

enum RshInitDMA::Control

List of additional flags.

Use flags combined by 'OR' statement to specify additional options in RshInitDMA::control field.

Enumerator
StandardMode 

No special options used.

This flag can be omited and used only for more readable code.
This flag meaning is: no differential mode, no frame frequency mode.

DiffMode 

Turn on differential mode.

One can check if device supports this mode or not by using RSH_CAPS_DEVICE_DIFFERENTIAL_INPUT_MODE.
See device user manual for details about analog input pin connect in two work modes.

FrameMode 

Turn on frame sampling mode.

When this flag is active, RshInitDMA::frequencyFrame field value is used as sampling rate inside frame.

MulSwitchStart 

Switch multiplexor on start.

Deprecated:
Not used at this moment, can be excluded from future releases.
enum RshInitDMA::DmaMode

Data acquisition modes list.

Values from this list are mutually exclusive (that is, use only one mode). Set it to the RshInitDMA::dmaMode field.

Enumerator
Single 

Single buffer mode.

So called "Start-Stop" mode.
Only one buffer of specified size is acquired, and then data transfer process is terminated.

See Also
RSH_CAPS_SOFT_GATHERING_IS_AVAILABLE
Persistent 

Continuous mode.

In this mode data acquisition and transfer will continue forever, until IRshDevice::Stop() method is called.

See Also
RSH_CAPS_SOFT_PGATHERING_IS_AVAILABLE

U32 RshInitDMA::dmaMode

Data acquisition mode.

One of the elements of RshInitDMA::DmaMode enum. There are only two modes actually: "Start-stop" mode and continuous mode.
One can use RSH_CAPS_SOFT_GATHERING_IS_AVAILABLE and RSH_CAPS_SOFT_PGATHERING_IS_AVAILABLE caps to check if device supports work in this two modes.

Examples:
FrameFrequencyMode_EN.cpp, IRshDevice_Get_RSH_GET_WAIT_GATHERING_COMPLETE_EN.cpp, IRshDevice_Init_RshInitDMA_EN.cpp, LA-1.5PCI_Persistent.cpp, LA-20USB_Persistent.cpp, LA-2M5PCI_Persistent.cpp, LA-2USB_Persistent.cpp, LA-50USB_Persistent.cpp, LA-5Net_Persistent.cpp, LA-i24USB_Persistent.cpp, LA-n150PCI_Persistent.cpp, and Leonardo2_Persistent.cpp.
