Detailed Description

Information about device digital ports.

This structure is used as container for obtaining port information.
One need pass this structure to IRshDevice::Get() method with parameter RSH_GET_DEVICE_PORT_INFO.

Examples:
LA-1.5PCI_Port.cpp, LA-20USB_Port.cpp, LA-2M5PCI_Port.cpp, LA-2USB_Port.cpp, LA-50USB_Port.cpp, LA-5Net_Port.cpp, LA-i24USB_Port.cpp, LA-n150PCI_Port.cpp, LA-n20-12PCI_Port.cpp, LA48DPCI_Port.cpp, Leonardo2_Port.cpp, and Python_Port.cpp.

Member Data Documentation

RSH_BUFFER_PORTINFO RshBoardPortInfo::confs

Configuration ports.

List of device configuration ports. Use RshBufferType::Size() field to determine, how many configuration ports device have.

Remarks
Most devices do not have configuration ports, only data ports.
Examples:
LA-1.5PCI_Port.cpp, LA-20USB_Port.cpp, LA-2M5PCI_Port.cpp, LA-2USB_Port.cpp, LA-50USB_Port.cpp, LA-5Net_Port.cpp, LA-i24USB_Port.cpp, LA-n150PCI_Port.cpp, LA-n20-12PCI_Port.cpp, LA48DPCI_Port.cpp, Leonardo2_Port.cpp, and Python_Port.cpp.
RSH_BUFFER_PORTINFO RshBoardPortInfo::ports

Data ports.

Actual digital ports, that can be used to input and output data. Use RshBufferType::Size() field to determine, how many data ports device have.

Examples:
LA-1.5PCI_Port.cpp, LA-20USB_Port.cpp, LA-2M5PCI_Port.cpp, LA-2USB_Port.cpp, LA-50USB_Port.cpp, LA-5Net_Port.cpp, LA-i24USB_Port.cpp, LA-n150PCI_Port.cpp, LA-n20-12PCI_Port.cpp, LA48DPCI_Port.cpp, Leonardo2_Port.cpp, and Python_Port.cpp.
