Detailed Description

Base initialization structure for ADC devices.

This structure contains parameters, common for all ADC devices.
It is never used in real device libraries, just as parent for RshInitDMA and RshInitMemory structures.

See Also
RshInitDMA | RshInitMemory

Member Enumeration Documentation

enum RshInitADC::ControlSynchro

List of additional synchronization flags.

Use flags combined by 'OR' statement to specify additional options in RshInitDMA::controlSynchro field.

Enumerator
FrequencySwitchOff 

Sampling rates for history and prehistory are equal.

Default option, work for all devices. SDK1 analog is ADC_CONTROL_ESW constant.

See Also
RSH_CAPS_DEVICE_FREQUENCY_SWITCH_PREHISTORY
SlopeFront 

Use signal front for synchronization.

This field is only actual when synchonization mode is active (see RshInitDMA::startType).
Mutually exclusive with RshInitDMA::SlopeDecline.

SlopeDecline 

Use signal decline for synchronization.

This field is only actual when synchonization mode is active (see RshInitDMA::startType).
Mutually exclusive with RshInitDMA::SlopeFront.

FrequencySwitchToMinimum 

Sampling rates for history and prehistory are different (switch to low frequency).

This flag can be used only for devices that support this feature ( RSH_CAPS_DEVICE_FREQUENCY_SWITCH_PREHISTORY). When flag is active, sampling rate is changed to low frequency (F = Fmax/8) when synchronization event occures. As a result, prehistory and main data will have different sampling rate.

SDK1 analog is ADC_CONTROL_FSW constant.

FrequencySwitchToMaximum 

Sampling rates for history and prehistory are different (switch to high frequency).

This flag can be used only for devices that support this feature ( RSH_CAPS_DEVICE_FREQUENCY_SWITCH_PREHISTORY). When flag is active, sampling rate is changed to high frequency when synchronization event occures. As a result, prehistory and main data will have different sampling rate.

enum RshInitADC::StartType

Start mode list.

This enum entities can be used to specify how to start ADC process.
To do so, set one of them as value for RshInitADC::startType field.

See Also
Synchronization
Enumerator
Program 

Program start.

Conversation will start as soon as command will be acquired by the device.
That means, ADC will start as soon as IRshDevice::Start() method is called.

Remarks
"As soon" is not immediately, because there is always some delays for functions calls, data transfer and device responce.
Timer 

Timer start.

Start data acquisition process using internal timer.

Deprecated:
This flag is not used in any library at the moment. In SDK1 some devices used Program mode, some Timer mode for the same thing. May be this flag will be removed in the future.
External 

Start data acquisition on external trigger.

Use external trigger input as source for data acquisition start command.

Remarks
There are two different mechanisms that can be activated by this flag: using external trigger input (see RSH_CAPS_DEVICE_SYNCHRO_EXTERNAL) or external start input (see RSH_CAPS_DEVICE_EXTERNAL_START).
Internal 

Internal synchronization.

Use analog input channels as source for trigger to start ADC.

Remarks
One can check if device supports this mode using RSH_CAPS_DEVICE_SYNCHRO_INTERNAL.
Usually, this feature is used by "Memory" devices.
See Also
RshChannel::Synchro | RshInitMemory::threshold
FrequencyExternal 

Use external frequency source for sampling.

External frequency will be used for sampling ADC. See RSH_CAPS_DEVICE_EXTERNAL_FREQUENCY for more details.

Master 

Use master device as start trigger source.

This flag may be used in systems where two or more devieces are started synchronously from one source. One device usually is master device, and all other are slave devices.

See Also
RSH_CAPS_DEVICE_SLAVE_MASTER_SWITCH

Additional synchonization options.

Combination of flags from RshInitDMA::ControlSynchro enum.
By default this parameter is equal to zero, which corresponds to following flag combination: RshInitDMA::FrequencySwitchOff | RshInitDMA::SlopeFront

double RshInitADC::threshold

Synchronization threshold.

Trigger threshold in volts. This parameter is actual, when device working in synchronization mode (RshInitDMA::startType is set to RshInitDMA::Internal or RshInitDMA::External).
When sygnal will bypass threshold, synchronization event will be triggered, and device start acquire and transfer data.

Remarks
When synchronization is turned off, this field is ignored.
