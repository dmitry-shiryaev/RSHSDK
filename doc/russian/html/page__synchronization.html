Синхронизация

Данная статься относится в большей степени к высокочастотным устройствам сбора данных с собственной памятью (см. RSH_CAPS_SOFT_INIT_MEMORY). Данные устройства, как правило, имеют дополнительные возможности для запуска преобразования по какому-либо событию. Настройка параметров сбора данных, и, в том числе, параметров синхронизации осуществляется посредством структуры инициализации RshInitMemory.

Запуск преобразования

Выбирая способ запуска устройства (поле RshInitMemory::startType) и комбинируя его с другими параметрами (предыстория, задержка старта, гистерезис) можно получить различные варианты работы устройства. Временные диаграммы работы в различных режимах приведены на рисунке:

  • В обычном режиме (RshInitMemory::startType == RshInitMemory::Program) сбор данных начинается непосредственно после получения команды IRshDevice::Start().
    Нужно учитывать, что запуск происходит не мгновенно, т.к. после вызова команды Start формируется сответствующая управляющая команда, которая должна быть отправлена в устройство, получена, распознана, и обработана его прошивкой. Задержка зависит от операционной системы, интерфейса устройства (PCI, USB), и других факторов.
  • В режиме синхронизации (RshInitMemory::External, RshInitMemory::Internal или RshInitMemory::Master) при получении команды IRshDevice::Start() устройство будет переведено в режим ожидания, и запись данных начнется только при получении события синхронизации.
    Выйти из режима ожидания (если, например, событие синхронизации не приходит слишком долго) можно, вызвав метод IRshDevice::Stop().
    Дополнительно, можно включить запись предыстории (размер которой можно варьировать), а также использовать задержку старта (длительность задается программно). Кроме того, можно комбинировать эти два параметра

Предыстория

Размер предыстории задается в поле RshInitMemory::preHistory как целое число от 0 до 15. Весь буфер (заданный объем памяти, RshInitMemory::bufferSize делится на 16 частей, а RshInitMemory::preHistory определяет, сколько этих частей будет использоватья под предысторию, а сколько - под данные. Если размер предыстории равен нулю, весь буфер будет состоять из данных, записанных после прихода события синхронизации. Если же, например, задан размер предыстории 4, 4/16 буфера (четверть) будет занята предысторией, а оставшаяся часть - основными данными.

Как только приходит событие синхронизации, дописывается нужный размер (размер буфера за вычетом размера предыстории) и после этого данные отправляются из внутренней памяти устройства в компьютер. Чтобы узнать, поддерживает ли устройство возможность записи предыстории, нужно проверить флаг RSH_CAPS_DEVICE_PREHISTORY.

Задержка старта

Задержка старта позволяет отсрочить запуск сбора данных относительно момента срабатывания синхронизации. Это полезно, если нужно, к примеру, отследить отклик на какое-то воздействие, который значительно (не хватит буфер чтобы записать весь сигнал) отстоит по времени от этого воздействия. Задать задержку старта можно в поле RshInitMemory::startDelay. Данная возможность поддерживается не всеми устройствами. Необходимо проверить наличие флага возможностей RSH_CAPS_DEVICE_START_DELAY.

Гистерезис

Используя поле RshInitMemory::hysteresis можно добавить "гистерезис" в схему поиска точки срабатывания синхронизации (точка перехода). Данная опция актуальна только при использовании цифровой синхронизации - как правило, данная схема реализована при использовании в качестве источника синхронизации аналоговых входов.

Без использования гистерезиса любой переход через заданный порог вызовет событие синхронизации. Это может привести к ложным срабатываниям, например из-за "дребезга" оцифровки, когда значение скачет вверх-вниз на один или два МЗР. Включив же гистерезис (для этого нужно задать в поле RshInitMemory::hysteresis значение больше нуля, в диапазоне 1-20), можно добиться четкого срабатывания: сигнал должен не только пересечь пороговый уровень, но и превысить дополнительный (как раз и определяемый заданным значением гистерезиса) порог. Если, допустим, сигнал привысил порог, но так и не достиг уровня порог+гистерезис, события синхронизации не произойдет.
Чтобы узнать, поддерживает ли устройство возможность задания гистерезиса нужно проверить флаг RSH_CAPS_DEVICE_HYSTERESIS.
